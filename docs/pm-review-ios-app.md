# 知合 Mosaic iOS App — 产品体验评审报告

> 评审日期：2026-02-24
> 评审版本：v1.0.0
> 评审设备：iPhone 17 模拟器 (iOS 26.2)
> 后端版本：FastAPI (localhost:8011)

---

## 一、总体印象

知合 Mosaic 是一款围绕「四笔钱」框架设计的家庭资产管理 App，覆盖了活钱、稳钱、长钱（基金组合）和保险四大板块。整体功能完整度较高，UI 风格简洁清晰，信息层级合理。但在细节体验、信息呈现、交互一致性和功能深度上仍有较大提升空间。

**体验评分：6.5 / 10**

---

## 二、逐页面分析与改进建议

### 1. 首次启动 — 服务器配置页

![服务器配置](pm-review-screenshots/01-initial-launch.png)

**现状**：首次打开只有一个极简的 IP + 端口输入表单，没有任何品牌标识、欢迎语或引导说明。

**问题**：
- P0：没有应用 Logo 或品牌标识，用户不知道自己在用什么 App
- P1：缺少使用说明，普通用户不理解「服务器地址」意味着什么
- P1：「测试连接」和「保存」是两个独立按钮，应合并流程：保存时自动测试
- P2：端口号默认 8011 已预填，但 IP 地址字段为空，无默认值

**改进建议**：
1. 顶部添加 App Logo + 名称「知合 Mosaic」+ 一句话简介
2. 增加引导文案说明为何需要配置服务器（本地部署架构）
3. 合并为一个「连接并保存」按钮，连接成功后自动进入主页
4. 考虑支持局域网自动发现（mDNS/Bonjour）

### 2. 总览（Dashboard）

![总览](pm-review-screenshots/02-dashboard.png)

**现状**：展示家庭总资产、四个桶的摘要卡片、资产走势图、待办提醒。

**问题**：
- P0：待办提醒区域被 Tab Bar 遮挡，只能看到「待办提醒」标题，实际内容需要滚动但页面滚动体验差
- P1：四桶卡片缺少各桶的占比信息，只有绝对金额，无法直观感知配置是否合理
- P1：走势图 Y 轴标签太密（¥0.00, 100.00, 200.00, 300.00），单位不直观，应显示为「万」
- P2：总资产卡片的盈亏只显示了长钱部分的盈亏，但标题是「家庭总资产」，容易造成误解
- P2：卡片之间间距过大，导致一屏信息密度偏低
- P2：缺少「上次更新时间」的显示

**改进建议**：
1. 每个桶卡片增加占比标签（如「活钱 62%」）
2. 走势图金额使用万元单位，优化 Y 轴刻度
3. 缩小卡片间距，让待办提醒区域能在首屏可见
4. 添加数据更新时间戳
5. 增加下拉刷新的视觉反馈动画

### 3. 四笔钱（入口列表）

![四笔钱](pm-review-screenshots/04-buckets-list.png)

**现状**：6 个导航入口按 3 个 Section 分组：活钱/稳钱、长钱三个子页面、保险。

**问题**：
- P1：纯列表导航缺少数据概览，用户需要进入每个子页面才能看到金额
- P1：「长钱」拆分成 3 个入口（组合概览 / 持仓明细 / 仓位管理）对普通用户不够直观，信息架构偏技术化
- P2：每个入口只有标题和副标题，没有关键数据摘要
- P2：列表项高度偏大，浪费屏幕空间

**改进建议**：
1. 每个入口行右侧增加关键金额数据（如 活钱 → ¥149.99万）
2. 考虑将「长钱」的三个入口合并为一个入口，内部用 Tab/Segment 切换
3. 或将此页面改为带金额摘要的卡片式布局，提供更多信息密度

### 4. 活钱

![活钱](pm-review-screenshots/05-liquid-list.png)

**现状**：顶部摘要卡片（总金额 + 预估年收益），下方资产列表。

**问题**：
- P1：「现金」出现了两条（¥986,879.49 和 ¥350,000.00），虽来自同一平台（招行），用户可能困惑为何分开
- P1：列表中没有显示年化收益率，只有「朝朝宝」显示了「年化 1.29%」，其余「现金」/「公积金」没有
- P2：所有项目左侧缺少平台 Logo 或类型图标区分
- P2：没有按平台分组或排序的选项

**改进建议**：
1. 为「活期存款」类型添加默认收益率（如 0.20%），而非留空
2. 增加平台图标或色标以区分不同来源
3. 支持按平台/金额排序

### 5. 稳钱

![稳钱](pm-review-screenshots/06-stable-list.png)

**现状**：3 笔定期存款，显示金额、年化利率、到期日。

**问题**：
- P1：到期日显示为纯文字（如「2027-01-05 到期」），缺少到期状态标签（已到期/即将到期/正常）
- P1：「特色整存整取」的到期日是 2026-09-25，距今不到 8 个月，但没有提醒标识
- P2：没有显示剩余天数
- P2：缺少到期后的操作建议

**改进建议**：
1. 添加到期状态彩色标签（红色=已到期，橙色=30天内到期，绿色=正常）
2. 在到期日旁显示「剩余 X 天」
3. 已到期的项目高亮提醒，并提供「续存」操作入口

> 注：代码中已有到期日标签逻辑，但截图中未显示，可能是样式不够明显或数据未触发。

### 6. 长钱 · 组合概览

![组合概览](pm-review-screenshots/07-growth-overview.png)
![组合概览下半部分](pm-review-screenshots/07b-growth-overview-bottom.png)

**现状**：四宫格统计（总盈亏、收益率、总市值、仓位）+ 走势图 + 平台分布 + 资产配置饼图。

**问题**：
- P0：仓位 41.40% 低于目标下限 50%，但此页面没有任何警告提示（仅在 Dashboard 待办提醒中有）
- P1：走势图数据量少（仅 4 个数据点），显示效果差，折线图形态不自然
- P1：资产配置饼图「其他 79.95%」占绝对多数，说明配置分类粒度太粗，缺乏实际参考价值
- P2：平台分布只显示了平台名称（如 alipay, 博时 app），名称不规范，应统一为中文
- P2：四宫格中「仓位」与「总盈亏」的颜色语义不一致 — 仓位偏低应该用警告色而非中性色

**改进建议**：
1. 仓位偏低/偏高时在四宫格卡片中加上警告图标和颜色
2. 平台名称统一规范化（alipay → 支付宝，博时 app → 博时基金）
3. 优化资产配置分类，减少「其他」占比，增加「混合型/QDII/指数型」等基金类型维度
4. 走势图在数据不足时显示提示文案而非稀疏折线

### 9. 仓位管理

![仓位管理](pm-review-screenshots/10-position.png)

**现状**：仪表盘（当前仓位 41.40%）+ 预算/资金卡片 + 投资策略列表。

**问题**：
- P1：仪表盘视觉上 41.40% 处于绿色区域，但实际低于目标下限 50%，颜色语义矛盾
- P1：「获取策略建议」按钮没有说明会产生什么结果，用户不确定要不要点
- P2：「简单仓位策略」和「资产再平衡策略」的描述文字太长，排版不够精炼
- P2：可用资金 ¥878,929.59 的来源不明确（是活钱？还是总预算减去已投资？）

**改进建议**：
1. 仓位低于目标下限时，仪表盘颜色应为橙/红色警告色
2. 「获取策略建议」增加简短说明（如「基于当前仓位生成调仓建议」）
3. 标注可用资金的计算逻辑
4. 策略卡片精简描述，重点突出「已启用/未启用」状态

### 10. 保险

![保险](pm-review-screenshots/11-insurance.png)

**现状**：摘要卡片（保单数/覆盖人数/年缴保费）+ 按被保人分组的保单列表。

**问题**：
- P1：保单信息密度偏高，单卡片展示太多字段，视觉层级不够清晰
- P1：「下次缴费」日期没有距今天数，且颜色太淡不够醒目
- P2：「有效」状态标签颜色与整体主题不协调
- P2：缺少保障覆盖范围的总览（如：重疾覆盖 X 人，医疗覆盖 X 人）

**改进建议**：
1. 保单卡片重新排版，主信息（名称+状态）一行，次要信息（保费+缴费日）一行
2. 缴费日临近时（30天内）用醒目颜色和倒计时标签
3. 增加保障覆盖概览（各险种的覆盖人数和保额）

### 11. 资产诊断

![资产诊断](pm-review-screenshots/12-diagnosis.png)

**现状**：直接渲染后端 JSON 诊断报告，以纯文本形式展示「诊断结论」。

**问题**：
- P0：诊断页是整个 App 中体验最差的页面。纯文本 JSON 结构化渲染，没有任何视觉设计
- P0：「Scan1 Mpt」等英文术语直接暴露给用户，无中文翻译
- P0：严重程度标签（high/medium）用纯文本而非视觉元素呈现
- P1：没有「重新诊断」功能，报告是静态的
- P1：缺少诊断评分/等级的总览性指标
- P2：信息架构混乱，所有诊断项平铺展示，没有分类

**改进建议**：
1. **整体重设计**：增加诊断评分（如 72 分 / 100），用颜色和图形直观展示
2. 按类别分组：集中度风险、地域风险、资产配置、仓位管理等
3. 每个诊断项用卡片展示：标题 + 严重程度标签（红/橙/绿）+ 描述 + 建议操作
4. 所有文案中文化，消除英文术语
5. 增加「立即诊断」按钮，可实时生成报告

## 三、跨页面系统性问题

### 3.1 信息架构

| 问题 | 优先级 | 说明 |
|------|--------|------|
| Tab 数量过多 | P1 | 5 个 Tab 中「诊断」和「数据」使用频率低，建议合并到「设置」或「更多」中 |
| 四笔钱入口层级深 | P1 | 用户需 2-3 次点击才能看到具体持仓，建议在列表项中展示关键数据 |
| 长钱拆分过细 | P2 | 概览/持仓/仓位 拆为 3 个入口，增加认知负担 |

### 3.2 视觉与交互一致性

| 问题 | 优先级 | 说明 |
|------|--------|------|
| 列表项样式不统一 | P2 | 活钱/稳钱/持仓/保险的列表卡片样式各异，缺少统一规范 |
| 金额格式不统一 | P2 | 有的显示完整小数（¥1,499,879.49），有的显示到整数，Dashboard 应使用万元单位 |
| 颜色语义不一致 | P1 | 仓位仪表盘绿色 ≠ 正常（实际低于目标），需统一颜色语义系统 |
| 空状态处理 | P2 | 部分页面有空状态但不确定是否所有 CRUD 页面都覆盖了 |

### 3.3 性能与可靠性

| 问题 | 优先级 | 说明 |
|------|--------|------|
| 基金详情加载慢 | P1 | 进入基金详情有明显的「加载中...」白屏时间，需优化加载体验 |
| 无离线支持 | P2 | 断网时无缓存数据可用，应缓存最近一次数据 |
| 数据过期无感知 | P1 | 净值数据已过期多天但 App 端无提示 |

## 五、总结

知合 Mosaic iOS App 在功能覆盖度上做到了四桶体系的完整呈现，CRUD 操作齐全，数据展示基本到位。但作为一个面向个人/家庭用户的理财管理工具，在以下三个维度仍有显著提升空间：

1. **信息密度与效率**：多处需要用户反复点击才能获取关键数据，应在入口处就展示核心指标
2. **视觉语义与一致性**：颜色、格式、布局需建立统一规范，尤其是警告/正常/异常状态的颜色语义
3. **诊断功能的产品化**：当前诊断页是 MVP 级别的 JSON 渲染，需要完整的产品设计来承载这个核心差异化功能

建议下一阶段重点投入诊断页面的重设计和 Dashboard 的信息密度优化，这两项改进将显著提升产品的核心价值感。
